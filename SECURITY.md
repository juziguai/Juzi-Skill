# 安全策略

## 支持范围

仅 stable 引用和最新正式市场 Release 属于受支持版本。main/beta 用于候选验证，不承诺兼容或持续可用。

## 报告漏洞

请通过 GitHub Private Vulnerability Reporting 或仓库所有者的私密渠道报告漏洞。不要在公开 Issue、PR、日志或截图中提交 Token、Cookie、请求正文、用户配置、本机私有路径或其他凭据。

报告至少包含：

- 受影响插件或纯 Skill；
- 可复现的最小步骤；
- 潜在影响和是否已经泄露；
- 已知的安全停止条件；
- 不包含真实秘密的验证证据。

## 处理边界

- 发现秘密后立即停止发布；先轮换秘密，再处理 Git 历史。
- 不直接编辑 Codex 插件缓存来“快速修复”。
- 公共仓库可见性变更前必须运行完整历史扫描和资产审计。
- 安全修复仍需经过源码验证、隔离安装、安装状态和新任务加载四层证明。

本项目不接受以破坏生产服务、绕过 UAC、读取真实请求正文或扩大数据访问范围为代价的安全验证。
