# 拆仓决策

Juzi-Skill 保持个人市场控制面和可安装发布包。子项目满足任一条件时进入拆仓评估：

- 规范源码超过 10 MiB 或单次克隆明显受大型资产影响；
- 拥有独立发布节奏、依赖树或多人协作边界；
- 测试时间使全局 CI 超过 15 分钟；
- 需要独立 Issue、Release、许可证或安全策略；
- 生成物与市场包无法通过确定性构建复原。

拆仓后：

1. 上游仓库保存规范源码和独立历史；
2. Juzi-Skill 只接收经过验证的发布包或确定 Git SHA；
3. .agents/projects.json 记录上游地址、版本和包哈希；
4. 不使用未固定分支或 Junction 把外部工作树直接暴露给市场。

当前旅游攻略因重复参考 HTML 接近主要体积来源，列为首个观察对象；尚未达到强制拆仓阈值。
