# 质量门禁

## 构建前

- [ ] 相对日期已转换为绝对日期，去返程和住宿晚数算术一致。
- [ ] `sampleOnly` 为 `false`，所有示例占位均已替换。
- [ ] 易变事实有 `verifiedAt`、状态边界和官方复核入口。
- [ ] 本地图片已压缩；无原始大 PNG、无第三方盗链、无未说明的 AI 示意图。
- [ ] 运行生成器 `--validate-only` 通过。

## 内容

- [ ] 交通、接驳、每日路线、票务、住宿、美食、预算、天气、行李和来源齐全。
- [ ] `days` 完整覆盖旅行日期，每日站点按 `HH:MM` 排序，天气日期逐日对齐。
- [ ] 每天节奏可执行，包含进站/登船/换乘缓冲和雨天兜底。
- [ ] 价格区间和余票不写成永久承诺。
- [ ] 用户要求保留旧文案时，对冻结基线执行可见文字逐段比较，缺失必须为 0。

## 网页

- [ ] 用户指定既有 HTML 时已记录原文件路径、字节数和 SHA256；原文件哈希在迭代前后保持一致，所有改动仅出现在派生文件。
- [ ] 迭代已有模板时保留原版配色、标题比例、章节顺序和卡片语言；新增模块不得抢占原有第一信息位，也不得插入并拆散原版连续信息组。
- [ ] 只有内联 CSS/JS；离线模式下无外部图片、字体或脚本依赖。
- [ ] 导航、清单、预算切换、天气刷新、返回顶部均可退化；正文不依赖联网。
- [ ] 行程驾驶舱在行前、旅行当天和行程结束后都有清晰状态；清单勾选会更新完成度并可刷新后恢复。
- [ ] 使用语义化标题、替代文本、可见键盘焦点、键盘可操作控件与 `prefers-reduced-motion`。
- [ ] 桌面四列/两列和移动单列布局无横向溢出；关键链接和按钮的可点区域至少 44px 高。
- [ ] 动效暂停和 `prefers-reduced-motion` 状态下，地图节点、路线列表和正文仍可见、可操作。
- [ ] 联网天气失败时，状态提示与天气卡均进入失败态，并保留至少一个官方复核入口。
- [ ] 图片与输出体积低于计划预算；两个副本存在时哈希一致。

```powershell
node "<skill-dir>\scripts\build-guide.mjs" --input ".\work\travel-plan.json" --validate-only
node "<skill-dir>\scripts\build-guide.mjs" --input ".\work\travel-plan.json" --output ".\outputs\旅游攻略.html"

node "<skill-dir>\scripts\refine-reference-guide.test.mjs"
node "<skill-dir>\scripts\refine-reference-guide.mjs" --check
```

前两条命令用于通用 JSON 生成路径；后两条只用于仓内厦门金标准原版。检查生成器报告的图片数量、总图片 KiB、HTML KiB 和 SHA256。若门禁失败，优化资产或补齐事实，不要直接放宽阈值。

## 交付说明

最终汇报包含：一句话结论、关键假设、核实日期、实时边界、输出文件、体积、验证结果和验收入口。不要把内部临时 JSON 或测试文件当成交付物。
